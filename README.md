# Микросервис авторизации с использованием JWT технологии
## Немного о сервисе:
Модель базы данных *User*

Pydantic-модели: *RegisterIn*, *LoginIn*, *UserOut*

`/register`
Основной функционал заключается в том, что пользователь регистрируется и его данные сохряняются в БД, пароль предварительно хешируется с помощью отдельной функции. 

`/login`
Основной функционал заключается в том, что пользователь вводит свои `логин` и `пароль`. Первым делом необъходимо проверить существования пользователя в БД, далее пароль с помощью отдельной функции проверяется на соответствие hash паролю. В последствии данные пользователя (а именно `username`) передаются в функцию `create_jwt`, где как раз уже и создаётся токен, посредством алгоритма шифрования. 

`/me`
Ручка, которая возвращает конкретную информацию об авторизированном пользователе
### Данный сервис упакован в Docker контейнер, что упрощает его запуск
### Инструкция для запуска
`docker-compose build` - создаём образ, от которого уже и будут строиться контейнеры.

`docker-compose up` - поднимаем контейнер с логами, чтобы убедиться, что всё хорошо.

`docker-compose down` - при необходимости можно удалить контейнер, обязательно без удаления томов, чтобы не потерять данные.
